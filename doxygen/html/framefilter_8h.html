A grand base class for framefilters.

With this base class, we achieve this kind of API:

FrameFilter ff1();
FrameFilter ff2();
SplitFrameFilter ff3();
FrameFilter ff4();
// Chaining simple framefilters (see SimpleFrameFilter child class)
ff1.cc(ff2).cc(ff3).cc(ff4)
// using split framefilters (see SplitFrameFilter child class):
ff1.cc(ff2).cc(ff3).cm(ff41,ff42,ff43) // cm == connect many
// continue with simple frame filters:
ff41.cc(ff5)
ff42.cc(ff6)
// also with split framefilters: connect various
ff3.cc(ff41)
ff3.cc(ff42)
// split framefilters: disconnect:
ff3.dc(ff41);
ff3.dc(ff42)
// framefilter callback cascade is started with:
ff1.go(frame);

WARNING: once filterchains have been connected to a thread, cc method can't be used, i.e. it is not thread safe. SplitFrameFilter is an exception to this as it is thread safe. Macro returning the name of the framefilter class
