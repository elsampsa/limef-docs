Detailed Description

CUDA converter subclass for DecodedToTensorFrameFilter.

Author
Sampsa Riikonen
Date
2025
Version
1.6.1

Expected input

DecodedFrame with av->format == AV_PIX_FMT_CUDA and a software format of AV_PIX_FMT_NV12 in hw_frames_ctx — i.e. frames produced by NVDEC. Any other format is rejected with a one-time error log and the converter deactivates (same behaviour as the CPU converters).

Output

TensorFrame with one GPU plane, shape (3, H, W), dtype UInt8. The kernel converts NV12 → planar RGB or BGR directly on the GPU — no CPU involvement and no intermediate packed buffer.

Split .cu / .cpp

The CUDA kernel and its launcher live in decoded_to_tensor_cuda.cu (compiled by nvcc, no spdlog). The converter class lives in decoded_to_tensor_cuda.cpp (compiled by g++, may use spdlog / FFmpeg headers).
