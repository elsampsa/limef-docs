Detailed Description

GLX/OpenGL video presenter: renders decoded frames on screen via GLX.

Subclass of PresenterThread that creates an X11 window with a GLX context and renders each decoded video frame using OpenGL. Supports three frame paths without any extra configuration:

  • CPU frames (SW decode, YUV420P / NV12): uploaded via glTexSubImage2D.
  • VAAPI frames (Intel/AMD GPU): CPU fallback via materializeToCPU(). (DMA-buf zero-copy requires EGL; use EGLPresenterThread for that path.)
  • CUDA frames (NVDEC, NV12 on GPU): zero-copy via cudaGraphicsGLRegisterImage
    • cudaMemcpy2DToArray (compiled only when LIMEF_CUDA is defined). This is the principal advantage of GLX over EGL on X11: NVIDIA's CUDA GL interop queries glXGetCurrentDisplay() internally and only works with a current GLX context. Requirement: the GLX context and the CUDA device must be on the same physical GPU. On hybrid (PRIME/optimus) systems the default GLX context is on the Intel/Mesa GPU; cudaGraphicsGLRegisterImage will fail and fall back to CPU upload. Force the NVIDIA GPU with: __NV_PRIME_RENDER_OFFLOAD=1 __GLX_VENDOR_LIBRARY_NAME=nvidia and set PresenterContext::bypass_compositor = true so KWin does not try to re-composite the NVIDIA-rendered window through its Mesa compositor. Alternatively, use sudo prime-select nvidia to make NVIDIA the system default (see tools/nvidia/check_display.bash).

GLX vs EGL

GLXPresenterThread EGLPresenterThread
CUDA zero-copy (X11) Yes (cudaGraphicsGLRegisterImage) No (CPU fallback)
VAAPI zero-copy No (CPU fallback) Yes (DMA-buf → EGLImage)
Wayland native No (X11-only) Yes (via XWayland / future wl path)

Use GLXPresenterThread when CUDA/NVDEC zero-copy is required on X11. Use EGLPresenterThread when VAAPI zero-copy is required or when Wayland compatibility matters.

Shaders:

  • NV12 → RGB (used for CUDA frames, 2-plane: Y as GL_R8, UV as GL_RG8)
  • YUV420P → RGB (used for CPU SW-decode frames, 3-plane: Y/U/V as GL_R8 each)

Example usage:

GLXPresenterThread presenter("glx", FrameFifoContext(false, 20, 40), 1280, 720);
mediafile.getOutput().cc(decoding_ff).cc(presenter.getInput());
presenter.start();
// ...
presenter.stop();
