All pool/device references live here — TensorFrame stays a clean data container.

Behaviour by gpu_target

HWAccel::SW (default) — CPU stack frames:

  • CPU source → CPU stack frame (base class memcpy, no override)
  • GPU source → CPU stack frame (D2H cudaMemcpy download) ← explicit download

HWAccel::CUDA — GPU stack frames:

  • CPU source → GPU stack frame (H2D cudaMemcpy upload — saves the extra D2D that UploadToGPUFrameFilter + fifo would do)
  • GPU source → GPU stack frame (D2D copy, delegated to base)

Non-TensorFrames (SignalFrame, StreamFrame, …) always pass through via clone.
