Detailed Description

Thread for reading frames from USB cameras via V4L2.

Author
Sampsa Riikonen
Date
2025
Version
1.6.1

Overview

USBCameraThread reads raw frames from a USB camera device (e.g., /dev/video0) and sends them downstream as DecodedFrames. It is a producer thread similar to MediaFileThread0, but outputs decoded pixels instead of compressed packets.

Output Frames

  • StreamFrame: Sent once at startup with video dimensions, pixel format, fps
  • DecodedFrame: Continuous stream of pixel data (after SwScale conversion)

Internal Processing

USB cameras typically output YUYV (or MJPEG, not supported yet). An internal SwScaleFrameFilter converts to the target format (default NV12) which is suitable for GPU upload.

Pipeline Integration

Typical usage for GPU processing pipeline:

USBCameraThread -> UploadGPUFrameFilter -> GPUProcessingThread -> ...

Limitations

  • Currently only supports raw pixel formats (YUYV, etc.)
  • MJPEG cameras not yet supported (would need internal decoding)
  • Audio capture not implemented (USB camera mics are separate ALSA devices)

Future Work

  • MJPEG support via internal software decoder
  • Audio capture from associated ALSA device
