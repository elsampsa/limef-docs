Detailed Description

Base class for threads that present media at the correct wall-clock time.

PresenterThread receives DecodedFrames from upstream and holds them in an internal queue. Instead of processing each frame immediately on arrival, it inspects Frame::absolute_timestamp_ and presents frames only when their timestamp is "due" (i.e. wall-clock time >= frame timestamp).

Timing loop

  • When a new frame arrives it is pushed into pending_frames_ without being recycled. The mainLoop's task() call is then invoked and checks whether any frames in the queue are due.
  • task() presents every due frame (oldest first), recycles it, and returns the remaining time until the next frame is due. This value becomes the read() timeout so the thread wakes up at exactly the right moment.
  • Frames with an invalid timestamp (absolute_timestamp_ == 0) are considered immediately due and are presented on the next task() call.

Input frame type

PresenterThread is templated on DecodedFrame — only DecodedFrame objects are accepted by the input FrameFifo. Supported pixel formats depend on the subclass (GLXPresenterThread, EGLPresenterThread), but all expect a DecodedFrame as the unit of input.

When feeding from a TensorFrame pipeline, prepend TensorToDecodedFrameFilter before the presenter's getInput():

TensorToDecodedFrameFilter t2d("t2d");
t2d.cc(presenter.getInput());
// … upstream produces TensorFrames → t2d → presenter

Subclassing

Override present() to dump the frame to a display, audio device, etc. The default implementation logs the frame's timestamp to the thread logger.

Slot support

Frames carry a slot value (see Frame::getSlot()). Subclasses can use it to route frames from different sources to different surfaces inside a single PresenterThread instance.

Notes on frame ownership

Frames in pending_frames_ belong to the FrameFifo stack and must NOT be deleted manually. They are returned to the stack via framefifo->recycle() after present() returns.

Example usage:

PresenterThread presenter("presenter", FrameFifoContext(false, 10, 20));
producer.getOutput().cc(presenter.getInput());
presenter.start();
// ...
presenter.stop();

Present a single frame.

Called from task() when a frame's timestamp is due. The default implementation logs the frame info. Override in subclasses to write to a display / audio device / etc.

The frame must NOT be recycled inside this method — the caller (task()) handles recycling after present() returns.
