Detailed Description

A pass-through thread for TensorFrame streams.

Receives TensorFrames, copies them through its internal TensorFrameFifo (breaking the synchronous filterchain), and forwards them unchanged downstream via output_ff.

This is the building block for the CPUBlock and GPUBlock pipeline nodes. It establishes the thread boundary — i.e. the point at which frames are copied into a pre-allocated pool and ownership transfers to this thread — without performing any computation. Subclass and override processFrame() to add per-frame work.

The TensorFrameFifo honours the gpu_target field of the context:

  • HWAccel::SW (default) — keep frames on CPU
  • HWAccel::CUDA — upload incoming CPU frames to GPU in one H2D copy

Non-TensorFrame frames (SignalFrame, StreamFrame, etc.) are dispatched by the base class and pass through unchanged.

FrameFifoContext ctx;
TensorThread tr("worker", ctx);
upstream.cc(tr.getInput());
tr.getOutput().cc(downstream);
tr.start();
