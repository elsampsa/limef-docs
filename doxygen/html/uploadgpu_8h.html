Detailed Description

FrameFilter for uploading CPU frames to GPU memory.

Author
Sampsa Riikonen
Date
2025
Version
1.6.1

Overview

UploadGPUFrameFilter takes DecodedFrames with CPU pixel data and uploads them to GPU memory, producing DecodedFrames with hw_frames_ctx set.

GPU Backend Support

NVIDIA (CUDA) - Desktop & Jetson

  • Uses AV_PIX_FMT_CUDA
  • Direct integration with cv::cuda::GpuMat
  • Preferred for Jetson development

Intel (VAAPI)

  • Uses AV_PIX_FMT_VAAPI
  • Surface format typically NV12
  • OpenCV integration requires cl_intel_va_api_media_sharing extension

Memory Management

The filter owns its own hw_frames_ctx pool. When a frame is uploaded:

  1. Get buffer from pool (av_hwframe_get_buffer)
  2. Upload CPU data to GPU (av_hwframe_transfer_data)
  3. Pass GPU frame downstream

Downstream consumers (e.g., DecodedFrameFifo) should create their own pools and copy the GPU frame data, ensuring proper buffer ownership.

Input Requirements

  • DecodedFrame with CPU pixel data
  • Format should match hw_frames_ctx sw_format (typically NV12)
  • If format mismatch, upload may fail or require internal conversion
