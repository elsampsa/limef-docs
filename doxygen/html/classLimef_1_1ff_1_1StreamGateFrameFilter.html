Detailed Description

Passes only source-originated StreamFrames; drops relay replays.

Acts as a gate on the StreamFrame::from_source flag:

  • from_source == true → frame passes through unchanged.
  • from_source == false → frame is silently dropped.

All other frame types (PacketFrame, DecodedFrame, TensorFrame, SignalFrame, CodecFrame, …) pass through unconditionally.

When to use

Place a StreamGateFrameFilter immediately upstream of any stateful filter that must only reinitialise on genuine stream lifecycle events (source restart, resolution/format change) and must never see relay StreamFrames injected by intermediate routing filters such as SwitchFrameFilter.

This is the composable alternative to per-filter from_source checks. Use it when you want the relay-suppression behaviour to be explicit and visible in the pipeline topology rather than buried inside each consumer.

Example

StreamGateFrameFilter gate("gate");
SwitchFrameFilter sw("sw", 2);
EncodingFrameFilter enc("enc", params);
// relay StreamFrames from sw never reach enc
sw.cc(0, gate);
gate.cc(enc);

Relation to EncodingFrameFilter

EncodingFrameFilter already inspects from_source internally and skips reinitialisation for relay StreamFrames. StreamGateFrameFilter provides the same protection as an explicit pipeline element, useful when the downstream filter does not (yet) handle from_source itself.
