Detailed Description

TensorToDecodedFrameFilter — converts TensorFrame (c, h, w) → DecodedFrame.

Author
Sampsa Riikonen
Date
2025
Version
1.6.1

Handles both CPU and GPU (CUDA) input planes automatically — the path is selected per-frame based on Plane::isGPU().

Accepted input

TensorFrame with shape (3, H, W) or (1, H, W), dtype UInt8. Any other shape or dtype is logged as an error and the frame is dropped.

in_order describes the channel order of the input tensor:

  • ChannelOrder::RGB — tensor channels [R, G, B]
  • ChannelOrder::BGR — tensor channels [B, G, R]

CPU path (plane on CPU RAM)

Output: DecodedFrame with pixel format:

  • 3-channel input → AV_PIX_FMT_GBRP
  • 1-channel input → AV_PIX_FMT_GRAY8

GBRP is the natural inverse of GBRPConverter. Add SwScaleFrameFilter downstream to convert to YUV420P for software encoding if needed.

GPU path (plane on CUDA device, LIMEF_CUDA only)

Only 3-channel tensors are supported (NV12 is colour-only). Output: DecodedFrame with AV_PIX_FMT_CUDA / sw_format = AV_PIX_FMT_NV12. A launchCHWtoNV12 CUDA kernel converts the CHW device buffer directly into an NV12 CUDA surface — no CPU involvement. The internal CudaCHWConverter manages its own AVHWFramesContext pool (lazy-init, reinit on resize). The output CUDA DecodedFrame can be fed directly into an NVENC encoder.

StreamFrame

On the first frame and whenever dimensions or CPU/GPU mode change, a StreamFrame is emitted before the DecodedFrame. Downstream encoders use this to initialise. The StreamFrame format reflects the active path:

  • CPU path → AV_PIX_FMT_GBRP or AV_PIX_FMT_GRAY8
  • GPU path → AV_PIX_FMT_CUDA

Timestamps

abs_timestamp_ from the TensorFrame is converted to PTS (microseconds since first frame) using PTStamp. The DecodedFrame time_base is set to {1, 1000000}.

Why not pointer-wrap?

See md/tensorframe.md — short answer: FFmpeg encoders call av_frame_make_writable() which breaks with null buf[] refs. We just copy.
