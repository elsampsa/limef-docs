Expected usage

Prepend a SwScaleFrameFilter that normalises the pixel format to either AV_PIX_FMT_GBRP (colour) or AV_PIX_FMT_GRAY8 (greyscale). Any other format arriving here is rejected (one-time error log, filter deactivates).

Why GBRP and not packed RGB24?

GBRP is planar: each channel (G, B, R) lives in its own contiguous AVFrame plane. Converting to the output shape (C, H, W) is then three row-by-row memcpys in the desired channel order — no element-wise scatter needed. Packed RGB24 (interleaved RGBRGB…) would require de-interleaving, which is roughly 3× more memory traffic for the same result.

Future: direct format converters

A single-copy path (e.g. YUV420P → CHW directly, without the SwScale hop) is possible but would need SIMD (SSE/AVX/NEON) to match SwScale's throughput. Add such converters here only when profiling shows the two-copy path is a real bottleneck.
