Detailed Description

Routes incoming frames to exactly one of N output terminals.

Only the currently active terminal receives frames. Calling toggle() at runtime changes which terminal is active. When a terminal is first activated, any cached CodecFrame and StreamFrame metadata are replayed automatically before the next live frame is forwarded, so the new downstream has the codec/stream context it needs even if those one-time metadata frames have already passed.

Caching rules:

  • CodecFrame and StreamFrame are cached keyed by their slot number.
  • A newer frame for the same slot replaces the cached one.
  • All other frame types are never cached.

from_source flag on replayed StreamFrames

When a StreamFrame is replayed from the cache (i.e. sent as part of a toggle() activation rather than having arrived live from upstream), StreamFrame::from_source is set to false on the replayed copy before it is forwarded. This lets downstream filters distinguish a genuine stream lifecycle event (source restart, resolution change) from a cold-branch relay.

Downstream filters that maintain persistent state (e.g. EncodingFrameFilter) can skip reinitialisation when from_source == false. Alternatively, place a StreamGateFrameFilter immediately downstream to drop all relay StreamFrames before they reach stateful filters.

SwitchFrameFilter sw("sw", 3);
sw.cc(0, ff1);
sw.cc(1, ff2);
sw.cc(2, ff3);
sw.toggle(1); // route all frames to ff2
sw.toggle(-1); // drop all frames (no active terminal)

Activate output terminal index.

Pass -1 to deactivate all terminals (frames are silently dropped). When activating a terminal, a lazy replay of cached metadata is scheduled: the next call to go() will forward all cached CodecFrame / StreamFrame entries before the live frame.
