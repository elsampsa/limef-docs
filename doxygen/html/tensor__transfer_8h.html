Both filters are only available when LIMEF_CUDA is defined.

When to use these vs TensorFrameFifo

TensorFrameFifo with gpu_target=CUDA or gpu_target=SW performs the same H2D / D2H transfer at the thread boundary in a single copy — the preferred approach whenever a thread boundary already exists.

Use these filters when you need an explicit mid-chain transfer without a thread boundary (e.g. upload once, then fan out to multiple GPU consumers via a SplitFrameFilter).

Pass-through rules

  • Non-TensorFrame frames always pass through unchanged.
  • A TensorFrame already on the target device passes through unchanged (no copy).
  • Empty TensorFrames (num_planes == 0) pass through unchanged.
